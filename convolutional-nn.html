In this formative assignment, I used the code provided in Unit 9, Machine Learning module, and applied changes to analyze another set.

See the code with commentary and outcomes.

The solution works for various K values (K stands for a number of final sets). Below find compared step 1 and step 14 of clustering for K=3 and K=4, respectively.


Due to a lucky random starting points sampling, there was no much efforts left to perform. Still, the program was able to balance the numbers of points between two lower sets.


In this example where K=4, the algorithm faced some more complicated starting give-away, however also demonstrated an ability to balance between two neighbouring sets.


Another example illustrates the shortcoming of the simplest version of K-means algoritm: getting stuck in local minumum. When two, roughly equivalent group of points try to "tug-of-war" the center of mass point, it oscillates there-and-back-again never resulting in a dominace of any part.
